# 小說寫作規範 (Writing Rules)

**目的**：確保輸出的內容可以直接複製貼上到小說連載網站，無需二次格式化。

## 1. 格式要求 (Format)
*   **純文本 (Plain Text)**：嚴禁使用任何 Markdown 語法。
    *   ❌ 不要使用 `**粗體**`
    *   ❌ 不要使用 `# 標題`
    *   ❌ 不要使用 `* 列表`
    *   ❌ 不要使用 `> 引用`
    *   ❌ 不要使用 `[]` 或 `()` 包裹系統提示（除非是小說內容一部分，但也儘量用中文全角符號）
*   **系統提示/面板**：
    *   系統面板內容使用【】或 [] 包裹，保持純文本格式。
    *   例如：【系統提示：檢測到高危險代碼植入。】
*   **分段**：
    *   段落之間空一行。
    *   對話需單獨成段。

## 2. 字數要求 (Word Count)
*   每章節正文（不含標點符號）需 **3000 字以上**。

## 3. 內容結構 (Structure)
*   **章節標題**：第一行直接寫章節名（例如：第十六章 投名狀）。
*   **結尾**：章節結束時標註（本章完）。

## 4. 寫作風格 (Style)
*   **沉浸感**：減少說明性文字，多用描寫和動作推進。
*   **系統感**：主角的系統視角描寫要清晰，但不要過度依賴數據堆砌，除非是關鍵爽點。
